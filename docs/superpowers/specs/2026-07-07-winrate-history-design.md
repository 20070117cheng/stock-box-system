# 過去一年每日勝率（勝率歷史）設計

日期：2026-07-07
狀態：已與使用者確認

## 目的

時光機一次只能看一天。使用者想看「過去一年裡，每個選股日的名單後來勝率如何」
與整年平均，作為策略是否有優勢的整體證據。

## 使用者已確認的決策

1. 主指標＝固定觀察窗 3 個月（A 版）：每個選股日往後追蹤 3 個月結算，
   日與日可比、歷史凍結；最近 3 個月的日子標「發展中」、不計入年平均
2. 輔指標＝算到今天（B 版）：當「目前戰況」看，可切換顯示
3. 兩版來自同一次回測（回測跑到今天，途中在窗結算日snapshot 一次）
4. 預先計算：GitHub Actions 每日排程算好存 repo，網頁直接讀，秒開
5. 網頁可用「目前的停利停損」重算（沿用快取名單，只重跑回測，等 1～2 分鐘）
6. 門檻或 KD 參數改變會使歷史名單失效：改 config.json 預設後，
   隔日排程自動重建整份快取（線上不支援改門檻重算）

## 計算定義

- 交易日清單：DB 內過去 365 天有資料的日期
- 每個選股日 d：用 scan(as_of=d) 取名單（同參數算過就快取，永不重算）
- 每檔股票：load_prices(d－1年, 今天) → run_backtest(起算日=d)，跑到今天
  - A 版：取「d＋3 個月」當天（或之前最後一個交易日）那列的損益
    - 若今天還沒到窗結算日 → 該選股日標「發展中」
  - B 版：取最後一列損益
  - 有進場＝該時點前持股張數曾 > 0；未進場者不計入勝率
- 每日勝率＝獲利檔數 ÷ 有進場檔數（無人進場＝無值）
- 年平均勝率＝已結算（非發展中）日子的勝率平均

## 檔案與模組

- `core/config.py`：新增 `winrate_window_months: 3`
- `core/history.py`（新模組）：
  - `trading_dates(conn, start, end)`：DB 的交易日清單
  - `update_scan_cache(conn, cfg, dates, cache_df, progress)`：補掃快取缺的日期
  - `compute_winrates(conn, cfg, scans_df, today, progress)`：逐日逐檔回測，
    回傳每日勝率表（A/B 兩版欄位）
- `jobs/daily.py`：每日流程最後多一步「更新勝率歷史」，可 `--skip-history` 跳過
  - 產出 `outputs/history/scans.parquet`（名單快取，含選股日欄）、
    `outputs/history/winrates.parquet`、`meta.json`（記錄快取用的參數）
  - meta 內選股參數（門檻/KD）與現行 config 不符 → 整份快取重建
  - 快取只保留過去約一年，舊的裁掉
- `app/main.py` 時光機分頁頂部新增「過去一年每日勝率」區塊：
  - 年平均勝率（A 版、排除發展中）等摘要指標
  - 折線圖：x=選股日、y=勝率；發展中的日子用淺色標示；
    勾選框切換顯示 B 版（算到今天）虛線
  - 「用目前停利停損重算」按鈕：讀快取名單＋DB 重跑回測（有進度條），
    結果存 session_state，不寫回 repo

## 成本估算

- 首次回填：約 250 個交易日全市場掃描（本機實測每日約 9 秒）＋
  約數千次單檔回測，Actions 一次跑 1～2 小時內，公開 repo 免費
- 之後每天：新增 1 天掃描＋全部日子重跑回測（B 版每天變）約 10～20 分鐘

## 測試

- pytest：快取補缺與重用（不重掃已有日期）、勝率計算（造一檔必勝一檔必敗的
  假資料驗證勝率 50%）、發展中判定、未進場不計入
- 既有測試全過；本機真實 DB 抽樣驗證後才推
