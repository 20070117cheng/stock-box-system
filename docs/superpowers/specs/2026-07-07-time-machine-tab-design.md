# 時光機分頁設計（Time Machine Tab）

日期：2026-07-07
狀態：已與使用者確認

## 目的

現有系統的回測有事後選擇偏誤：名單用「今天接近 3 年高點」選出，回頭看過去 6 個月
的模擬損益天生偏正，無法回答「這套策略歷史上有沒有優勢」。

時光機分頁讓使用者站在過去任一天，用**當時**的資料跑選股（`scan(as_of=...)`），
再往後模擬到指定截止日，觀察「入選後實際發生什麼」。選股當下不知道未來，
結果才可信。

## 使用者已確認的決策

1. 總表 + 單檔詳細圖，兩者都要
2. 觀察截止日可調，預設今天（放到今天）
3. 掃描當場現算 + 快取（第一次等 1～3 分鐘，有進度條），不預先算
4. 策略參數獨立放在分頁內，與側欄（現在策略）互不影響
5. 賣出時點由策略動態觸發（回測引擎內建），截止日只是觀察窗終點；
   到期仍持有者以截止日收盤計未實現損益
6. 全站移除所有 emoji

## 版面

新增第三個分頁「時光機」，與「每日選股」「個股分析」並列。

### 控制區（分頁內，獨立參數）

- 選股日期（date_input）
- 觀察截止日（date_input，預設今天；需晚於選股日期，否則擋下並提示）
- 移動停利 %、固定停損 %、距 3 年高點門檻 %、每次加碼張數
  （預設值取 config.json，調整只影響本分頁）
- 「執行掃描」按鈕，按下才開始計算

### 區塊一：績效總表

摘要指標列：入選檔數、有進場檔數、獲利／虧損檔數、勝率、平均報酬 %、合計損益。

- 回測引擎從空手開始觀察，進場條件（突破箱頂＋KD 多頭＋週低點抬高）觸發才買，
  可能整段都沒進場 → 狀態「未進場」，不計入勝率
- 勝率與平均報酬只統計有進場的股票

表格每檔一列：代號、股名、選股日收盤、期末狀態（未進場／持有中／已出場）、
最後出場日、報酬 %、損益金額、期末說明（進出場原因）。

### 區塊二：單檔詳細

- 從名單下拉選一檔
- 與「個股分析」同款雙圖（股價＋箱頂箱底＋停利停損線＋進出場記號、下方 KD），
  抽成共用函式避免兩份程式碼
- 圖上加垂直虛線標「選股日」
- 圖表範圍：選股日前 6 個月 ～ 觀察截止日
- 附回測每日明細 expander

## 技術設計

### core/scanner.py

`scan()` 加選用參數 `progress: Callable[[int, int], None] | None = None`，
迴圈內回報 (已處理檔數, 總檔數)。預設 None，每日排程與既有測試不受影響。

### app/main.py

- 移除全部 emoji（page_icon、標題、分頁名、按鈕、expander 標題）
- 抽共用函式 `render_stock_chart(...)`：現有個股分析的雙圖程式碼搬入，
  新增選用參數 `mark_date`（畫垂直虛線）；兩個分頁共用
- 時光機掃描快取：`@st.cache_data`，key =（選股日、門檻、KD 參數），
  進度條參數用底線前綴排除雜湊
- 總表：對名單每檔 `load_prices(start=選股日-1年, end=截止日)` →
  `run_backtest(prices, 選股日, cfg_tm)`，取最後一列彙整
- 掃描結果與總表存 `st.session_state`，切換單檔不重算

### 錯誤處理

- 截止日 ≤ 選股日：`st.error` 擋下
- 選股日太早（資料不足）：scan 自然回傳較少檔或空名單，顯示提示
- 該日無任何入選：`st.warning`

### 測試

- 新增 pytest：`scan(progress=...)` 有被呼叫且 (done, total) 遞增
- 既有 29 個測試維持全過
- 本機用真實 DB 跑一次「過去日期掃描 → 名單回測到截止日」腳本驗證
- Streamlit 本機起服務，確認三個分頁渲染正常、無 emoji 殘留
